# Signal Array — superseded docs

This folder contains documents from the original venture-scale plan for Signal Array that have been **superseded by the reorientation** captured in [`../ideation/00-REORIENTATION.md`](../ideation/00-REORIENTATION.md).

The reorientation (17 May 2026) shifted Signal Array from "open-core commercial SaaS, building toward a venture-grade synthetic UX research business" to "calibration-disciplined research methodology, deployed as bursty consulting engagements." The docs in this folder describe the SaaS-business plan that is no longer the operational picture.

## What's in here

| File | What it is | Why it's dormant |
|---|---|---|
| `Signal Array – Business Plan.md` | Three-year customer-ramp plan, hiring trajectory, funding posture | Venture-scale ambition retired in favor of solo-operator consulting model |
| `Signal Array – Marketing & GTM Plan.md` | Three sales motions, funnel KPIs, 12-month lead-gen content calendar | No active sales motion or pipeline tracking under the reorientation |
| `Signal Array – Financial Model.xlsx` | Monthly P&L projections across base / conservative / optimistic / downside scenarios | The 3-year customer-ramp model no longer reflects what's being built |

## When to revisit

These docs become potentially relevant again if any of the reactivation triggers in *00-REORIENTATION* are met:

- Three or more unsolicited inbound commercial inquiries within ~60 days of Moloco-001 publishing or CHI 2027 submission
- A Moloco contract extension that explicitly funds engineering capacity
- A credible co-builder partner (co-founder, sales partner, research lab) approaching with a concrete proposal
- A personal-life change that meaningfully expands the time budget

Until one of those is met, treat these docs as **historical reference**, not as plans to execute.

## Do not act on these without re-evaluating

The metrics, commitments, hiring triggers, and revenue targets in these files were written before the reorientation. Acting on them directly would re-introduce exactly the SaaS-operations load the reorientation was designed to remove. If you find yourself reading one of these docs and thinking about executing something from it, **first re-read `../ideation/00-REORIENTATION.md` and confirm the action is consistent with the current frame.**
